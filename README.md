# Bilibili 首页视频数据标注（简洁版）

> 在 B 站首页每个视频卡片下方直接显示点赞率和投币率，一目了然判断视频质量。

![版本](https://img.shields.io/badge/版本-2.0.0-blue)
![许可](https://img.shields.io/badge/许可-MIT-green)

## 📖 简介
你是否还在为刷不到高质量B站视频而烦恼？刷了一堆视频发现内容质量都很低？没事儿，这个脚本就是为了解决这个问题的。
这是一个**油猴脚本（UserScript）**，专为PC端 B 站首页设计。它会自动获取每个视频的详细数据，并在视频卡片下方显示：

- 👍 **点赞率** = 点赞数 ÷ 播放量
- 🪙 **投币率** = 投币数 ÷ 播放量

当数据达到"优质视频"标准时，会以**粉色高亮**显示。

| 特性 | 说明 |
|------|------|
| 🎯 **简洁直观** | 直接在视频卡片下显示数据，无需悬停或点击 |

| 🎨 **视觉反馈** | 优质数据粉色高亮，普通数据灰色显示 |

## ⚙️ 配置说明

脚本顶部有一个 `CONFIG` 对象，可以根据需要调整：

```javascript
const CONFIG = {
    LIKE_RATIO_THRESHOLD: 0.05,   // 点赞率阈值，默认 5%
    COIN_RATIO_FACTOR: 0.5,       // 投币率系数，默认为点赞率的 50%
    API_DELAY: 200,               // 请求间隔，单位毫秒
    CONCURRENT_LIMIT: 3,          // 最大并发请求数
    CACHE_TTL: 10 * 60 * 1000,    // 缓存时间，默认 10 分钟
    DEBUG: false,                 // 调试模式
};
```

### 配置项详解

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `LIKE_RATIO_THRESHOLD` | `0.05` | 点赞率 ≥ 5% 视为优质 |
| `COIN_RATIO_FACTOR` | `0.5` | 投币率 ≥ 点赞率 × 50% 视为优质 |
| `API_DELAY` | `200` | 每次 API 请求后等待 200ms |
| `CONCURRENT_LIMIT` | `3` | 最多同时发起 3 个请求 |
| `CACHE_TTL` | `600000` | 缓存有效期 10 分钟（600000ms）|
| `DEBUG` | `false` | 设为 `true` 可在控制台查看日志 |

## 📊 质量判断标准

### 点赞率判断

```
点赞率 = 点赞数 ÷ 播放量

如果 点赞率 ≥ 5%  →  显示粉色（优质）
否则              →  显示灰色（普通）
```

### 投币率判断

```
投币率 = 投币数 ÷ 播放量
投币率阈值 = 当前视频的点赞率 × 50%

如果 投币率 ≥ 投币率阈值  →  显示粉色（优质）
否则                      →  显示灰色（普通）
```

**为什么投币率是动态阈值？**

投币比点赞成本更高（需要消耗硬币），所以标准相对降低。同时，一个高点赞率的视频，其投币率标准也会相应提高。

## 🔧 技术实现

### 工作流程

```
1. 脚本启动
      │
      ▼
2. 扫描页面上的视频卡片
      │
      ▼
3. 提取每个视频的 BVID
      │
      ▼
4. 检查缓存是否有数据
      │
      ├─ 有缓存 → 直接使用
      │
      └─ 无缓存 → 调用 B 站 API 获取
                     │
                     ▼
5. 计算点赞率和投币率
      │
      ▼
6. 在视频卡片下方显示数据
      │
      ▼
7. 监听页面变化，处理新加载的视频
```

### 核心技术点

| 技术 | 用途 |
|------|------|
| `GM_xmlhttpRequest` | 跨域调用 B 站 API |
| `MutationObserver` | 监听 DOM 变化，处理懒加载 |
| `WeakSet` | 记录已处理的卡片，避免重复 |
| 并发队列 | 控制 API 请求频率，防止限流 |
| 缓存机制 | 减少重复请求，提升性能 |

### 使用的 B 站 API

```
GET https://api.bilibili.com/x/web-interface/view?bvid={BVID}

返回数据结构:
{
  "code": 0,
  "data": {
    "stat": {
      "view": 播放量,
      "like": 点赞数,
      "coin": 投币数,
      ...
    }
  }
}
```

## ❓ 常见问题

### Q: 为什么有些视频显示"加载中"很久？

**A:** 可能原因：
1. 网络延迟
2. API 请求队列中等待处理
3. 该视频数据获取失败

### Q: 为什么看不到数据标注？

**A:** 请检查：
1. 油猴扩展是否已启用
2. 脚本是否已启用
3. 是否在 B 站首页（`www.bilibili.com`）
4. 尝试刷新页面

### Q: 会不会被 B 站封号？

**A:** 不会。脚本只是读取公开数据，没有任何违规操作。且有以下保护机制：
- 请求间隔 200ms
- 最多 3 个并发请求
- 10 分钟缓存避免重复请求

### Q: 如何卸载脚本？

**A:**
1. 点击油猴图标
2. 进入"管理面板"
3. 找到脚本，点击删除
